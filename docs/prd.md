## 1. 제품 개요

### 1.1 문제 정의

LLM을 도입하려는 기업과 개발자가 폭발적으로 늘고 있다. 그러나 실제 도입 과정에서 공통적으로 겪는 문제가 있다.

- **선택의 과부하:** 상용·오픈소스 합쳐 수백 개의 모델이 존재하고, 매주 신규 모델이 출시된다. 어떤 모델이 우리 업무에 맞는지 판단할 기준이 없다.
- **비용 예측 불가:** 모델마다 가격 체계가 다르고, 프롬프트 캐싱·Batch API·양자화 등 비용 절감 옵션이 다양하지만 이를 종합적으로 시뮬레이션할 수 없다.
- **산업 특화 기준 부재:** 범용 벤치마크(MMLU, GPQA 등)는 존재하지만, "고객 상담 챗봇에 최적인 모델"이나 "법률 문서 분석에 강한 모델"처럼 업무 맥락에 맞는 비교 기준이 없다.
- **인프라 정보 분산:** 오픈소스 모델을 셀프호스팅하려면 GPU 스펙, VRAM, 서빙 프레임워크 정보가 필요하지만 모델마다 흩어져 있다.
- **조합 설계의 부재:** 실무에서는 단일 모델이 아니라 "분류는 저가 모델, 응답은 고급 모델"처럼 멀티모델 라우팅이 필요한데, 이를 설계할 도구가 없다.

### 1.2 제품 비전

> **어떤 산업, 어떤 업무든 — "우리 상황에 맞는 LLM 조합"을 10분 안에 찾고, 비용까지 시뮬레이션할 수 있는 플랫폼.**

단순한 모델 스펙 비교를 넘어, 사용자의 산업·업무·예산·인프라 조건을 입력하면 최적의 모델 조합과 운영 전략을 제안하는 의사결정 도구를 만든다.

### 1.3 목표 사용자

| 사용자 | 핵심 니즈 | 주요 사용 기능 |
|--------|-----------|---------------|
| 사업 기획자 / PM | 업무별 최적 모델 선정, 도입 비용 근거 확보 | 산업별 추천, 비용 시뮬레이터, 비교 리포트 |
| 개발자 / MLOps | 모델 스펙 비교, 셀프호스팅 인프라 설계, API 연동 판단 | 인프라 가이드, VRAM 계산기, 벤치마크 비교 |
| 경영진 / 의사결정자 | 요약된 비교 근거, 비용 전망, ROI 판단 | 리포트 내보내기, 비용 차트, 라우팅 절감 효과 |
| 프리랜서 / 1인 개발자 | 예산 내 최적 모델 빠른 탐색 | 가격순 정렬, 가성비 랭킹, 로컬 실행 가이드 |

### 1.4 경쟁 환경과 차별점

| 기존 도구 | 한계 | 우리의 차별점 |
|-----------|------|--------------|
| Artificial Analysis | 벤치마크·속도 비교 중심. 업무 맥락 없음 | 산업·업무별 적합도 점수 + 라우팅 전략 제안 |
| OpenRouter Rankings | 토큰 사용량 랭킹만 제공 | 비용 시뮬레이터 + 셀프호스팅 손익분기점 |
| PricePerToken | 가격 비교 전용 | 인프라 스펙 + 한국어 등 다국어 평가 통합 |
| 각 벤더 공식 문서 | 자사 모델만 다룸 | 36개+ 모델 크로스벤더 통합 비교 |

## 2. 기능 요구사항

### F1. 모델 데이터베이스

**F1.1 모델 정보 구조**

각 모델은 다음 정보를 포함한다.

- 기본 정보: 모델명, 제공사, 유형(상용/오픈소스), 출시일, 라이선스
- 아키텍처: 전체 파라미터, 활성 파라미터, 구조(Dense/MoE), 컨텍스트 윈도우
- 비용: Input/Output 토큰 단가, 캐싱 할인율, Batch API 할인율
- 범용 평가: 응답 품질, 속도, 추론력, 코딩, 멀티모달 (각 100점)
- 다국어 평가: 한국어, 영어, 일본어, 중국어 등 주요 언어별 점수
- 산업별 적합도: 업무 유형별 적합도 점수 (F6 참조)
- 벤치마크 원점수: MMLU, GPQA, SWE-bench, AIME, HLE, MGSM 등
- 인프라 요구사항: 최소 GPU 구성, VRAM(FP16/INT4), 권장 프레임워크, 예상 t/s
- 메타: 특이사항 메모, 데이터 출처 URL, 최종 갱신일

**F1.2 데이터 관리**

- 관리자 웹 UI에서 모델 CRUD
- 각 데이터에 "최종 확인일" 표시
- 가격 변경 시 이력 보존
- 향후 외부 API 자동 수집 연동을 고려한 구조

### F2. 모델 탐색

**F2.1 테이블 뷰**

- 전체 모델 한 화면 조회
- 필터: 유형, 파라미터 티어, 제공사, 라이선스, 가격 범위, 언어별 점수 하한
- 모든 수치 컬럼 정렬
- 모델명·제공사 텍스트 검색
- 컬럼 표시 커스터마이즈

**F2.2 파라미터 사이즈별 카드 뷰**

- 오픈소스 모델을 티어별 그룹핑 (초대형/대형/중형/소형/초소형)
- 각 카드에 인프라 요구사항 강조
- 그룹 헤더에 해당 티어 공통 하드웨어 가이드

**F2.3 모델 상세 페이지**

- 전체 스펙, 벤치마크 시각화, 인프라 가이드, 비용 예시
- 유사 모델 자동 추천 (같은 티어/가격대)
- 가격 변동 이력 차트
- 개별 URL로 검색엔진 인덱싱

---

### F3. 모델 비교

**F3.1 비교 선택**

- 테이블/카드에서 클릭으로 비교 목록 추가 (최대 4개)
- 비교 목록 페이지 이동 시 유지
- 플로팅 바로 현재 선택 모델 항시 표시

**F3.2 비교 뷰**

- 나란히 카드 배치
- 레이더 차트 (선택한 평가 항목 기준)
- 가격 차이 절대값 + 배수 표시
- 인프라 차이 하이라이트
- 항목별 우위 모델 시각 표시

**F3.3 공유·저장·내보내기**

- URL 쿼리 파라미터로 비교 상태 인코딩 → 링크 공유
- 로그인 사용자: 비교 세션 저장/불러오기
- PDF, PNG, CSV 내보내기
- 경영진 보고용 1페이지 요약 포맷

---

### F4. 비용 시뮬레이터

**F4.1 API 비용 계산**

입력:
- 일 평균 요청 건수
- 평균 Input/Output 토큰 수
- 월 운영 일수
- 프롬프트 캐싱 적용률 (슬라이더)
- Batch API 활용 비율 (슬라이더)

출력:
- 선택 모델들의 월간/연간 비용 바 차트
- 파라미터 변경 시 실시간 갱신

**F4.2 셀프호스팅 손익분기점**

입력:
- GPU 임대 시간당 단가 (또는 구매 가격 + 감가)
- 일 가동시간
- 월 부대비용 (전기, 관리 인건비)

출력:
- "API vs 셀프호스팅" 월간 비용 비교
- 요청량 증가에 따른 손익분기점 차트
- "일 N건 이상이면 셀프호스팅이 유리" 메시지

**F4.3 라우팅 비용 시뮬레이션**

- 단일 모델 처리 vs 다단계 라우팅 처리 비용 비교
- 라우팅 비율 조절 (예: 70% 저가, 20% 중급, 10% 고급)
- 라우팅 적용 시 절감률 자동 계산

**F4.4 내보내기**

- CSV, PDF, PNG
- 경영진 보고용 1페이지 요약

### F5. 인프라 가이드

**F5.1 VRAM 계산기**

- 입력: 파라미터 수, 양자화 방식(FP16/INT8/INT4), 컨텍스트 길이
- 출력: 필요 VRAM + 적합 GPU 매핑
- MoE 모델 안내 (전체 가중치 로드, 활성 파라미터만 연산)

**F5.2 GPU 레퍼런스**

- 주요 GPU별 VRAM, 가격, 특성 테이블
- 역방향 조회: "이 GPU로 돌릴 수 있는 모델 목록"

**F5.3 서빙 프레임워크 가이드**

- vLLM, Ollama, llama.cpp, SGLang 비교 (특성, 적합 상황, 장단점)
- 모델별 권장 프레임워크 매핑

### F6. 산업·업무별 추천

**F6.1 산업별 적합도 평가 체계**

모델에 산업·업무 유형별 적합도 점수를 부여한다. 점수는 해당 업무에 중요한 역량(한국어, 속도, 추론력, 컨텍스트 길이, 비용 등)에 가중치를 두어 산출한다.

산업/업무 프리셋:

| 카테고리 | 업무 유형 | 핵심 가중 요소 |
|----------|-----------|---------------|
| 고객 서비스 | CS 챗봇, FAQ 자동 응답 | 속도, 다국어, 비용 |
| 고객 서비스 | 복합 문의 처리, 에스컬레이션 | 품질, 추론력, 다국어 |
| 영업/마케팅 | 이메일·메시지 자동 생성 | 품질, 다국어, 비용 |
| 영업/마케팅 | 리드 스코어링, 고객 세그먼트 | 추론력, 분석력 |
| 영업/마케팅 | 콘텐츠 생성 (블로그, SNS) | 품질, 창의성, 다국어 |
| 개발/IT | 코드 생성, 리뷰, 디버깅 | 코딩, 추론력 |
| 개발/IT | 문서화, 기술 문서 작성 | 품질, 지시따르기 |
| 개발/IT | 에이전트 워크플로우 | 추론력, 도구사용, 속도 |
| 금융 | 리포트 분석, 리스크 평가 | 추론력, 정확도, 보안 |
| 금융 | 고객 상담 (은행/보험) | 다국어, 규정준수, 속도 |
| 법률 | 계약서 검토, 판례 분석 | 추론력, 컨텍스트, 정확도 |
| 법률 | 법률 문서 초안 작성 | 품질, 지시따르기 |
| 의료/헬스케어 | 의료 상담 보조, 증상 분류 | 정확도, 안전성, 다국어 |
| 의료/헬스케어 | 의학 문헌 분석 | 추론력, 컨텍스트 |
| 교육 | 학습 튜터, 문제 풀이 | 추론력, 다국어, 비용 |
| 교육 | 교육 콘텐츠 생성 | 품질, 창의성 |
| 이커머스 | 상품 추천, 리뷰 분석 | 속도, 비용, 다국어 |
| 이커머스 | 상품 설명 자동 생성 | 품질, 비용 |
| 물류/제조 | 데이터 분석, 이상 감지 | 추론력, 속도 |
| 공공/행정 | 민원 응대, 정책 문서 요약 | 다국어, 정확도, 보안 |
| 미디어/콘텐츠 | 기사 요약, 번역 | 품질, 다국어, 속도 |
| 연구/R&D | 논문 분석, 실험 설계 보조 | 추론력, 컨텍스트, 정확도 |

- 관리자가 프리셋의 가중치를 조정할 수 있다
- 사용자도 자기 업무에 맞게 가중치를 커스터마이즈할 수 있다

**F6.2 정적 추천 (시나리오별)**

- 위 프리셋별로 사전 정의된 추천 모델 조합을 제공한다
- 각 시나리오에 상용 추천, 가성비 추천, 오픈소스 추천을 포함한다

**F6.3 동적 추천**

사용자 입력:
- 산업 선택 (단일)
- 업무 유형 선택 (다중)
- 우선순위 정렬 (드래그 앤 드롭): 품질, 비용, 속도, 다국어, 보안 등
- 월 예산 상한
- 셀프호스팅 가능 여부 + 보유 GPU
- 주요 사용 언어

출력:
- 가중 점수 기반 상위 5개 모델 추천
- 점수 산출 근거 투명 공개
- 해당 모델 조합의 월간 예상 비용

**F6.4 라우팅 전략 제안**

- 입력된 업무 유형에 맞게 "분류→일반→복잡" 다단계 모델 조합 제안
- 각 단계 적합 모델 + 예상 비용 표시
- 단일 고급 모델 대비 절감률 계산

**F7.1 사용량 랭킹**

### F8. 사용자 기능

**F8.1 즐겨찾기**

- 관심 모델 별표 추가
- 즐겨찾기 필터 뷰
- 비로그인 시 로컬 저장, 로그인 시 서버 동기화

**F8.2 비교 세션 관리**

- 이름 붙여 저장, 목록 조회, 불러오기

**F8.3 내보내기**

- 테이블: CSV

**F8.4 URL 공유**

- 필터, 비교 선택, 시뮬레이터 설정을 URL에 인코딩

## 5. 데이터 구조

### 5.1 핵심 엔터티

```
모델
├── 식별: id, 이름, slug, 제공사, 유형, 티어
├── 아키텍처: 전체 파라미터, 활성 파라미터, 구조, 컨텍스트 윈도우, 라이선스
├── 비용: input 단가, output 단가, 캐싱 할인율, batch 할인율
├── 범용 평가: 품질, 속도, 추론력, 코딩, 멀티모달 (각 0~100)
├── 다국어 평가: { ko, en, ja, zh, ... } (각 0~100)
├── 벤치마크: { mmlu, gpqa, swe_bench, aime, hle, mgsm, ... }
├── 인프라: { 최소GPU, vram_fp16, vram_int4, 권장프레임워크, 예상tps }
├── 메타: 출시일, 메모, 출처URL[], 최종갱신일, 색상코드

산업 프리셋
├── id, 카테고리(산업), 업무 유형명
├── 가중치 맵: { 품질: 0.3, 속도: 0.2, 비용: 0.2, 다국어: 0.2, 추론: 0.1, ... }
├── 추천 모델 목록 (정적 추천용)
├── 설명 텍스트

가격 이력
├── model_id, input 단가, output 단가, 기록일

즐겨찾기
├── user_id, model_id

비교 세션
├── user_id, model_id[], 제목, 생성일
```

### 5.2 점수 산출 로직

산업별 적합도 점수 = Σ (모델의 항목별 점수 × 해당 프리셋의 가중치)

예시 — "CS 챗봇" 프리셋:

```
적합도 = (속도 × 0.25) + (한국어 × 0.25) + (비용역수 × 0.20) + (품질 × 0.20) + (추론 × 0.10)
```

사용자 커스터마이즈 시 가중치를 직접 조절하고, 결과가 실시간 반영된다.
